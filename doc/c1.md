# Capítulo 1: Apresentação do Projeto

## 1.1 Contexto e Enquadramento

Este documento apresenta o trabalho desenvolvido no âmbito da disciplina de **Desenvolvimento Web I** da Licenciatura em Informática na **Universidade da Maia (UMAIA)**, correspondente ao **Momento 3 de Avaliação Contínua** do ano letivo 2025/26.

O projeto consiste no desenvolvimento de um sistema completo de gestão de manutenção de automóveis, implementado com recurso a tecnologias modernas de desenvolvimento web e seguindo uma arquitetura baseada em serviços REST.

### 1.1.2 Objetivos Gerais

O objetivo principal deste trabalho é demonstrar a aplicação prática dos conhecimentos adquiridos na disciplina através do desenvolvimento de uma aplicação web completa, nomeadamente:

1. **Construir uma API REST** robusta e bem estruturada utilizando o framework **LoopBack 4**
2. **Desenvolver uma interface de backoffice** funcional com **React-Admin**
3. **Implementar um modelo de dados** relacional com múltiplas entidades e relações
4. **Aplicar boas práticas** de desenvolvimento web e arquitetura de software
5. **Containerizar a aplicação** utilizando **Docker**
6. **Documentar adequadamente** a API com OpenAPI 3.0

## 1.3 Âmbito do Projeto

### 1.3.1 Funcionalidades Incluídas

O sistema desenvolvido permite realizar as seguintes operações:

#### Gestão de Clientes

- Criar, editar e eliminar clientes
- Visualizar lista de clientes com pesquisa e filtros
- Associar múltiplos veículos a cada cliente
- Visualizar histórico de manutenções por cliente

#### Gestão de Veículos

- Registar novos veículos com informação detalhada
- Editar informação de veículos existentes
- Associar veículos a clientes
- Consultar histórico de manutenções por veículo

#### Gestão de Oficinas

- Cadastro de oficinas parceiras
- Informação de contacto e localização
- Gestão de agendamentos por oficina
- Capacidade e disponibilidade

#### Catálogo de Serviços

- Criar e manter catálogo de serviços oferecidos
- Definir preços e duração estimada
- Categorização de serviços (preventiva, correctiva, etc.)
- Descrições detalhadas

#### Gestão de Agendamentos

- Criar novos agendamentos
- Associar múltiplos serviços a um agendamento
- Definir datas e oficinas
- Estados de agendamento (pendente, confirmado, concluído, cancelado)

#### Histórico de Manutenções

- Registar manutenções realizadas
- Associar ao veículo correspondente
- Registar custos e observações
- Consultar histórico completo

## 1.5 Metodologia de Desenvolvimento

### 1.5.1 Abordagem Adotada

O desenvolvimento seguiu uma abordagem **Code-first**, conforme especificado no enunciado:

1. **LoopBack 4** para desenvolvimento expedito da API
2. **React-Admin** para desenvolvimento rápido da interface
3. **Iterações incrementais** com testes contínuos
4. **Versionamento com Git** desde o início

### 1.5.2 Ferramentas Utilizadas

#### Desenvolvimento

- **Visual Studio Code** - Editor de código principal
- **DataGrip / MySQL Workbench** - Gestão da base de dados
- **Postman** - Testes de API
- **Git** - Controlo de versões

#### Colaboração

- **GitHub** - Repositório e gestão de código
- **Discord/WhatsApp** - Comunicação da equipa

- **Docker Desktop** - Ambiente de desenvolvimento
- **Docker Hub** - Partilha de imagens
